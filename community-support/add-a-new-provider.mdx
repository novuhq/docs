# How to add a new provider?

Novu currently supports five channelsÂ `in_app`,Â `push`,Â `email`,Â `chat`Â andÂ `sms`. 

ForÂ `in_app`Â we support only our own provider, so new providers cannot be added to this channel. For the other four channels, we support the integration of external providers. This guide will help in adding new providers for any of these 4 channels.Â 

In this guide, we are adding a new provider for the email channel, but all of the mentioned steps are similar for other channels as well.

## Description

Providers allow us to handle message delivery over multiple channels. 

We have multiple providers for each channel (SMS, Email, Push and Chat). To get started with adding a new provider let's look at setting up our repository.

# Add config items to the list

In order to build the UI integration store, we need to provide it list of all provider integrations. This part is made up of two parts:

- Create credentials config
- Add ProviderId Enum
- Add provider configuration to the providers list

### 1. Create credentials config

Every provider requires some credentials to create an instance. Novu will add these credentials fields in the integration store provider's form so that users can use their credentials to connect to their preferred provider to use for that channel notification. For example, in the above addedÂ `example-provider`, we have only one credentialÂ `ApiKey`. We will need to add a config object forÂ `example-provider`Â with all existing provider's configs like below.

**libs/shared/src/consts/providers/credentials/provider-credentials.ts**

```tsx

  {
    key: CredentialsKeyEnum.ApiKey,
    displayName: "API Key",
    type: "text",
    required: true,
  },
  ...mailConfigBase,
];
```

1. Here theÂ `key`Â is of typeÂ `CredentialsKeyEnum`. If you want to add a new type of credentials key, you can add toÂ `CredentialsKeyEnum`.
2. `displayName`Â is a human-friendly easy to understand name which will be shown in the provider integration form for this credential.
3. `type`Â here means text field type. this can be a string for text, text for text-area, or a switch for the toggle.
4. `required`Â is of boolean type.
5. `mailConfigBase`Â is an object having default credentials required by anyÂ `email`Â provider. Make sure not to add duplicate providers which are already there inÂ `mailConfigBase`. In the case of another channel provider, we will use that channel config base in place ofÂ `mailConfigBase`.

> A credential can be made secret by adding inÂ ./secure-credentials.tsÂ file.

# Add provider handler in the API

### 1. Adding the provider dependency in the application-generic package

In the previous step, we created a standalone provider package that will be published to NPM. However currently in our development environment, it is not yet published. In order to use it locally please go to theÂ `package.json`Â located inÂ `apps/packages/application-generic/package.json`Â and add it manually to the dependencies list:Â `"@novu/<NEW_PROVIDER_NAME>": "^<VERSION>"`

> Please note that the provider name and version can be found from the providerÂ package.jsonÂ you created earlier.

After adding the dependency runÂ `pnpm run setup:project`Â from the root of the mono repo. so, it can create the required symlinks for the newly created package.

### 2. Create a provider handler

In order to map internally the different providersâ€™ credentials, we need to add a provider handler at the respective channel handlers located. For Email, it can be found atÂ `apps/packages/application-generic/src/factories/mail/handlers`. Other channel handlers can also be found here.

Create a new fileÂ `example-provider.handler.ts`Â here with the following code

**packages/application-generic/src/factories/mail/handlers/example-provider.handler.ts**

```tsx
import { ChannelTypeEnum } from "@novu/shared";
import { ExampleProviderEmailProvider } from "@novu/example-provider";
import { BaseHandler } from "./base.handler";

export class ExampleProviderHandler extends BaseHandler {
  constructor() {
    super("example-provider", ChannelTypeEnum.EMAIL);
  }

  buildProvider(credentials, from: string) {
    const config: { apiKey: string } = { apiKey: credentials.apiKey };

    this.provider = new ExampleProviderEmailProvider(config);
  }
}
```

Add this line given below to export this handler

**packages/application-generic/src/factories/mail/handlers/index.ts**

```tsx

```

The last step is to initialize the handler in the factory located inÂ `packages/application-generic/src/factories/mail/mail.factory.ts`
