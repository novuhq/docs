Novu does not expose a single **upsert** endpoint for workflows. To create or change a workflow programmatically, combine [create](/api-reference/workflows/create-a-workflow), [update](/api-reference/workflows/update-a-workflow), and [retrieve](/api-reference/workflows/retrieve-a-workflow). To push a workflow from Development to Production, use [sync a workflow](/api-reference/workflows/sync-a-workflow).

This recipe shows two patterns that behave like an upsert, plus standalone examples for each API call. The code samples use `payout-initiate` as a sample identifier only.

<Callout type="info">
Workflows are scoped to the environment. Create and edit them in **Development**, then [sync to other environments](/platform/developer/environments#publishable-assets) when you are ready.
</Callout>

You do not need to know a `workflowId` before you start. You choose it when you [create](/api-reference/workflows/create-a-workflow) a workflow. It can be any valid slug (letters, numbers, hyphens, dots, and underscores), such as `payout-initiate`, `weekly-digest`, or `order-shipped`.

- **Unique per environment:** The same `workflowId` can exist in Development and Production as separate workflows. Within one environment, each `workflowId` must be unique.
- **Immutable:** After creation, the identifier cannot be changed. Store the value you pick (config, env var, or your app constants) and pass it to update, retrieve, sync, and [trigger](/platform/workflow/trigger-workflow) calls.
- **Upsert flows:** On the first run, **create** sets the `workflowId`. On later runs, pass that same value so **update** and **sync** target the correct workflow.

The examples below use `payout-initiate` for consistency. Replace it with whatever identifier fits your product.

## Update a workflow first, create on 404 error

Try **update** first. If the workflow does not exist, the API returns **404**, then call **create**.

This is the preferred approach: one request when the workflow already exists, and no extra read before every write.

<Callout type="warn">
If two processes run at the same time against a missing workflow, both may get **404** and both may call **create**. The second create can return **409**. In that case, retry **update**.
</Callout>

<Tabs items={['Node.js', 'Python', 'cURL']}>
  <Tab value="Node.js">
    ```typescript
    import { Novu } from "@novu/api";
    import { z } from "zod";

    const novu = new Novu({
      secretKey: process.env.NOVU_SECRET_KEY!,
    });

    const WORKFLOW_ID = "payout-initiated";

    const payloadSchema = z.object({
      amount: z.number(),
      currency: z.string(),
      accountNumber: z.string(),
      accountName: z.string(),
      accountBank: z.string(),
    });

    const steps = [
      {
        type: "email" as const,
        name: "Send Payout Email",
        stepId: "payout-initiated-email",
        controlValues: {
          subject: "Payout Initiated",
          body: "Your payout {{payload.amount}} has been initiated. Please check your account {{payload.accountNumber}} for the details.",
          editorType: "html",
        },
      },
      {
        type: "in_app" as const,
        name: "Send Payout Inbox Message",
        stepId: "payout-initiated-in-app",
        controlValues: {
          subject: "Payout Initiated",
          body: "Your payout {{payload.amount}} has been initiated. Please check your account {{payload.accountNumber}} for the details.",
        },
      },
    ];

    const preferences = {
      user: {
        all: { enabled: true },
        channels: {
          email: { enabled: true },
          inApp: { enabled: true },
        },
      },
      workflow: {
        all: { enabled: true },
        channels: {
          email: { enabled: true },
          inApp: { enabled: true },
        },
      },
    };

    export async function upsertWorkflow() {
      const updatePayload = {
        name: "Payout Initiated",
        description: "This workflow is used to initiate a payout",
        tags: ["payout"],
        preferences,
        origin: "external" as const,
        validatePayload: false,
        payloadSchema,
        steps,
      };

      try {
        return await novu.workflows.update(updatePayload, WORKFLOW_ID);
      } catch (error: unknown) {
        const statusCode =
          error && typeof error === "object" && "statusCode" in error
            ? (error as { statusCode: number }).statusCode
            : undefined;

        if (statusCode === 404) {
          return await novu.workflows.create({
            ...updatePayload,
            workflowId: WORKFLOW_ID,
          });
        }

        if (statusCode === 409) {
          return await novu.workflows.update(updatePayload, WORKFLOW_ID);
        }

        throw error;
      }
    }
    ```

    ```python
    import os

    import novu_py
    from novu_py import Novu, models

    WORKFLOW_ID = "payout-initiated"

    PAYLOAD_SCHEMA = {
        "type": "object",
        "properties": {
            "amount": {"type": "number"},
            "currency": {"type": "string"},
            "accountNumber": {"type": "string"},
            "accountName": {"type": "string"},
            "accountBank": {"type": "string"},
        },
        "required": [
            "amount",
            "currency",
            "accountNumber",
            "accountName",
            "accountBank",
        ],
    }

    STEPS = [
        {
            "type": "email",
            "name": "Send Payout Email",
            "stepId": "payout-initiated-email",
            "controlValues": {
                "subject": "Payout Initiated",
                "body": "Your payout {{payload.amount}} has been initiated. Please check your account {{payload.accountNumber}} for the details.",
                "editorType": "html",
            },
        },
        {
            "type": "in_app",
            "name": "Send Payout Inbox Message",
            "stepId": "payout-initiated-in-app",
            "controlValues": {
                "subject": "Payout Initiated",
                "body": "Your payout {{payload.amount}} has been initiated. Please check your account {{payload.accountNumber}} for the details.",
            },
        },
    ]

    PREFERENCES = novu_py.PreferencesRequestDto(
        user=novu_py.UserWorkflowPreferencesDto(
            all=novu_py.WorkflowPreferenceDto(enabled=True),
            channels={
                "email": novu_py.ChannelPreferenceDto(enabled=True),
                "inApp": novu_py.ChannelPreferenceDto(enabled=True),
            },
        ),
        workflow=novu_py.PreferencesRequestDtoWorkflow(
            all=novu_py.WorkflowPreferenceDto(enabled=True),
            channels={
                "email": novu_py.ChannelPreferenceDto(enabled=True),
                "inApp": novu_py.ChannelPreferenceDto(enabled=True),
            },
        ),
    )


    def upsert_workflow() -> object:
        with Novu(secret_key=os.environ["NOVU_SECRET_KEY"]) as novu:
            update_dto = novu_py.UpdateWorkflowDto(
                name="Payout Initiated",
                description="This workflow is used to initiate a payout",
                tags=["payout"],
                preferences=PREFERENCES,
                origin=novu_py.ResourceOriginEnum.EXTERNAL,
                validate_payload=False,
                payload_schema=PAYLOAD_SCHEMA,
                steps=STEPS,
            )

            try:
                return novu.workflows.update(
                    workflow_id=WORKFLOW_ID,
                    update_workflow_dto=update_dto,
                )
            except models.NovuError as error:
                if error.status_code == 404:
                    return novu.workflows.create(
                        create_workflow_dto=novu_py.CreateWorkflowDto(
                            name=update_dto.name,
                            description=update_dto.description,
                            workflow_id=WORKFLOW_ID,
                            tags=update_dto.tags,
                            validate_payload=update_dto.validate_payload,
                            payload_schema=update_dto.payload_schema,
                            steps=update_dto.steps,
                            preferences=update_dto.preferences,
                        )
                    )

                if error.status_code == 409:
                    return novu.workflows.update(
                        workflow_id=WORKFLOW_ID,
                        update_workflow_dto=update_dto,
                    )

                raise
    ```

## Fetch a workflow first, then create or update

Call [retrieve a workflow](/api-reference/workflows/retrieve-a-workflow) first. If it returns **404**, **create**; otherwise **update**.

This costs an extra GET on every run but makes the branch explicit and avoids relying on update failure for control flow.

### Update a workflow

Returns **200** on success. Pass the workflow identifier as the path parameter (or SDK argument), not only in the body.

`preferences` and `origin` are required on update when managing workflows through the API.

<Tabs items={['Node.js', 'Python', 'cURL']}>
  <Tab value="Node.js">
    ```typescript
    import { Novu } from "@novu/api";
    import { z } from "zod";

    const novu = new Novu({
      secretKey: process.env.NOVU_SECRET_KEY!,
    });

    export const updateAWorkflow = async () => {
      return await novu.workflows.update(
        {
          name: "Payout Initiated",
          description: "This workflow is used to initiate a payout",
          tags: ["payout"],
    ```

### Publish a workflow

After you create or update a workflow in **Development**, publish it to another environment (typically Production) with [sync a workflow](/api-reference/workflows/sync-a-workflow). Pass the target environment identifier from the Novu Dashboard.

Use your Development secret key when calling sync. The workflow is copied to the target environment under the same `workflowId`, which is unique in that environment.

<Tabs items={['Node.js', 'Python', 'cURL']}>
  <Tab value="Node.js">
    ```typescript
    import { Novu } from "@novu/api";

    const novu = new Novu({
      secretKey: process.env.NOVU_SECRET_KEY!,
    });

    export const publishWorkflowToProduction = async () => {
      return await novu.workflows.sync(
        {
          targetEnvironmentId: "target-environment-identifier",
        },
        "payout-initiated"
      );
    };
    ```
  </Tab>
  <Tab value="Python">
    ```python
    import os

    from novu_py import Novu

    with Novu(secret_key=os.environ["NOVU_SECRET_KEY"]) as novu:
        res = novu.workflows.sync(
            workflow_id="payout-initiated",
            sync_workflow_dto={
                "target_environment_id": "target-environment-identifier",
            },
        )
    ```
  </Tab>
  <Tab value="cURL">
    ```bash
    curl -X PUT "https://api.novu.co/v2/workflows/payout-initiated/sync" \
      -H "Authorization: ApiKey ${NOVU_SECRET_KEY}" \
      -H "Content-Type: application/json" \
      -d '{
        "targetEnvironmentId": "target-environment-identifier"
      }'
    ```
  </Tab>
</Tabs>

See [Publish (Sync) a workflow](/api-reference/workflows/sync-a-workflow).

## Which pattern should be used?

| Approach | Requests when workflow exists | Best for |
| --- | --- | --- |
| **Update first** (recommended) | 1 (`PUT`) | When the workflow usually already exists |
| **Fetch first** | 2 (`GET` + `PUT`) | When you want explicit existence checks or simpler branching in scripts |

Both patterns rely on the same three endpoints. Pick one, define a `workflowId` on create and reuse it on later calls, and keep step `stepId` values consistent so Novu updates steps in place instead of treating them as new resources.
