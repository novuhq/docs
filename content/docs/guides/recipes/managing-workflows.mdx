Use the Novu Workflows API to manage workflows programmatically across environments. Create, update, and upsert workflows in Development, and publish to Production with [Sync a workflow](/api-reference/workflows/sync-a-workflow).

<Callout type="info">
Workflows are scoped to the environment. Create and edit them in Development, then [sync to other environments](/platform/developer/environments#publishable-assets) when you are ready.
</Callout>

You do not need to know a `workflowId` before you start. You choose it when you [create](/api-reference/workflows/create-a-workflow) a workflow. It can be any valid slug (letters, numbers, hyphens, dots, and underscores), such as `payout-initiated`, `weekly-digest`, or `order-shipped`.

- *Unique per environment:* The same `workflowId` can exist in Development and Production as separate workflows. Within one environment, each `workflowId` must be unique.
- *Immutable:* After creation, the identifier (workflowId) cannot be changed. Use the same workflowId for update, retrieve, sync, and [trigger](/platform/workflow/trigger-workflow) calls.
- *Stable management flow:* On the first run, create sets the `workflowId`. On later runs, pass that same value so update and sync target the correct workflow.

## Prerequisites

Install the SDK for your stack and set your secret key from the [Novu Dashboard](https://dashboard.novu.co):

    ```bash

    ```
  </Tab>
  <Tab value="Python (novu-py)">

  </Tab>

## Workflow API methods reference

### Create a workflow

Returns **201** on success. Set `workflowId` here to define the identifier for this environment. You choose the value; it does not need to exist beforehand.

<Tabs items={['Node.js', 'Python', 'cURL']}>
  <Tab value="Node.js">
    ```typescript
    import { Novu } from "@novu/api";
    import { z } from "zod";

    const novu = new Novu({
      secretKey: process.env.NOVU_SECRET_KEY!,
    });

    export const createAWorkflow = async () => {
      return await novu.workflows.create({
        name: "Payout Initiated",
        description: "This workflow is used to initiate a payout",
        workflowId: "payout-initiated",
        tags: ["payout"],
        payloadSchema: z.object({
          amount: z.number(),
          currency: z.string(),
          accountNumber: z.string(),
          accountName: z.string(),
          accountBank: z.string(),
        }),
        steps: [
          {
            type: "email",
            name: "Send Payout Email",
            stepId: "payout-initiated-email",
            controlValues: {
              subject: "Payout Initiated",
              body: "Your payout {{payload.amount}} has been initiated. Please check your account {{payload.accountNumber}} for the details.",
              editorType: "html",
            },
          },
          {
            type: "in_app",
            name: "Send Payout Inbox Message",
            stepId: "payout-initiated-in-app",
            controlValues: {
              subject: "Payout Initiated",
              body: "Your payout {{payload.amount}} has been initiated. Please check your account {{payload.accountNumber}} for the details."
            },
          },
        ],
      });
    };
    ```

### Update a workflow

Returns **200** on success. Pass the workflow identifier as the path parameter (or SDK argument). `preferences` and `origin` are required on update when managing workflows through the API.

<Tabs items={['Node.js', 'Python', 'cURL']}>
  <Tab value="Node.js">
    ```typescript
    import { Novu } from "@novu/api";
    import { z } from "zod";

    const novu = new Novu({
      secretKey: process.env.NOVU_SECRET_KEY!,
    });

    export const updateAWorkflow = async () => {
      return await novu.workflows.update(
        {
          name: "Payout Initiated",
          description: "This workflow is used to initiate a payout",
          tags: ["payout"],
          preferences: {
            user: {
              all: { enabled: true },
              channels: {
                email: { enabled: true },

              },
            },
            workflow: {
              all: { enabled: true },
              channels: {
                email: { enabled: true },
                inApp: { enabled: true },
              },
            },
          },
          origin: "external",
          validatePayload: false,
          payloadSchema: z.object({
            amount: z.number(),
            currency: z.string(),
            accountNumber: z.string(),
            accountName: z.string(),
            accountBank: z.string(),
          }),
          steps: [
            {
              type: "email",
              name: "Send Payout Email",
              stepId: "payout-initiated-email",
              controlValues: {
                subject: "Payout Initiated",
                body: "Your payout {{payload.amount}} has been initiated. Please check your account {{payload.accountNumber}} for the details.",

                subject: "Payout Initiated",
    ```

### Upsert a workflow

Novu does not expose a single endpoint or SDK method to upsert a workflow. To upsert a workflow, combine [create](/api-reference/workflows/create-a-workflow) and [update](/api-reference/workflows/update-a-workflow) in a single function.

<Tabs items={['Node.js', 'Python', 'cURL']}>
  <Tab value="Node.js">
    ```typescript
    import { Novu } from "@novu/api";
    import { z } from "zod";

    const novu = new Novu({
      secretKey: process.env.NOVU_SECRET_KEY!,
    });

    const WORKFLOW_ID = "payout-initiated";

    const payloadSchema = z.object({
      amount: z.number(),
      currency: z.string(),
      accountNumber: z.string(),
      accountName: z.string(),
      accountBank: z.string(),
    });

    const steps = [
      {
        type: "email" as const,
        name: "Send Payout Email",
        stepId: "payout-initiated-email",
        controlValues: {
          subject: "Payout Initiated",
          body: "Your payout {{payload.amount}} has been initiated. Please check your account {{payload.accountNumber}} for the details.",
          editorType: "html",
        },
      },
      {
        type: "in_app" as const,
        name: "Send Payout Inbox Message",
        stepId: "payout-initiated-in-app",
        controlValues: {
          subject: "Payout Initiated",
          body: "Your payout {{payload.amount}} has been initiated. Please check your account {{payload.accountNumber}} for the details.",
        },
      },
    ];

    const preferences = {
      user: {
        all: { enabled: true },
        channels: {
          email: { enabled: true },
          inApp: { enabled: true },
        },
      },
      workflow: {
        all: { enabled: true },
        channels: {
          email: { enabled: true },
          inApp: { enabled: true },
        },
      },
    };

    export async function upsertWorkflow() {
      const updatePayload = {
        name: "Payout Initiated",
        description: "This workflow is used to initiate a payout",
        tags: ["payout"],
        preferences,
        origin: "external" as const,
        validatePayload: false,
        payloadSchema,
        steps,
      };

      try {
        return await novu.workflows.update(updatePayload, WORKFLOW_ID);
      } catch (error: unknown) {
        const statusCode =
          error && typeof error === "object" && "statusCode" in error
            ? (error as { statusCode: number }).statusCode
            : undefined;

        if (statusCode === 404) {
          return await novu.workflows.create({
            ...updatePayload,
            workflowId: WORKFLOW_ID,
          });
        }

        if (statusCode === 409) {
          return await novu.workflows.update(updatePayload, WORKFLOW_ID);
        }

        throw error;
      }
    }
    ```

    ```python
    import os

    import novu_py
    from novu_py import Novu, models

    WORKFLOW_ID = "payout-initiated"

    PAYLOAD_SCHEMA = {
        "type": "object",
        "properties": {
            "amount": {"type": "number"},
            "currency": {"type": "string"},
            "accountNumber": {"type": "string"},
            "accountName": {"type": "string"},
            "accountBank": {"type": "string"},
        },
        "required": [
            "amount",
            "currency",
            "accountNumber",
            "accountName",
            "accountBank",
        ],
    }

    STEPS = [
        {
            "type": "email",
            "name": "Send Payout Email",
            "stepId": "payout-initiated-email",
            "controlValues": {
                "subject": "Payout Initiated",
                "body": "Your payout {{payload.amount}} has been initiated. Please check your account {{payload.accountNumber}} for the details.",
                "editorType": "html",
            },
        },
        {
            "type": "in_app",
            "name": "Send Payout Inbox Message",
            "stepId": "payout-initiated-in-app",
            "controlValues": {
                "subject": "Payout Initiated",
                "body": "Your payout {{payload.amount}} has been initiated. Please check your account {{payload.accountNumber}} for the details.",
            },
        },
    ]

    PREFERENCES = novu_py.PreferencesRequestDto(
        user=novu_py.UserWorkflowPreferencesDto(
            all=novu_py.WorkflowPreferenceDto(enabled=True),
            channels={
                "email": novu_py.ChannelPreferenceDto(enabled=True),
                "inApp": novu_py.ChannelPreferenceDto(enabled=True),
            },
        ),
        workflow=novu_py.PreferencesRequestDtoWorkflow(
            all=novu_py.WorkflowPreferenceDto(enabled=True),
            channels={
                "email": novu_py.ChannelPreferenceDto(enabled=True),
                "inApp": novu_py.ChannelPreferenceDto(enabled=True),
            },
        ),
    )


    def manage_workflow() -> object:
        with Novu(secret_key=os.environ["NOVU_SECRET_KEY"]) as novu:
            update_dto = novu_py.UpdateWorkflowDto(
                name="Payout Initiated",
                description="This workflow is used to initiate a payout",
                tags=["payout"],
                preferences=PREFERENCES,
                origin=novu_py.ResourceOriginEnum.EXTERNAL,
                validate_payload=False,
                payload_schema=PAYLOAD_SCHEMA,
                steps=STEPS,
            )

            try:
                return novu.workflows.update(
                    workflow_id=WORKFLOW_ID,
                    update_workflow_dto=update_dto,
                )
            except models.NovuError as error:
                if error.status_code == 404:
                    return novu.workflows.create(
                        create_workflow_dto=novu_py.CreateWorkflowDto(
                            name=update_dto.name,
                            description=update_dto.description,
                            workflow_id=WORKFLOW_ID,
                            tags=update_dto.tags,
                            validate_payload=update_dto.validate_payload,
                            payload_schema=update_dto.payload_schema,
                            steps=update_dto.steps,
                            preferences=update_dto.preferences,
                        )
                    )

                if error.status_code == 409:
                    return novu.workflows.update(
                        workflow_id=WORKFLOW_ID,
                        update_workflow_dto=update_dto,
                    )

                raise
    ```

For request details, refer to [Create a workflow](/api-reference/workflows/create-a-workflow) and [Update a workflow](/api-reference/workflows/update-a-workflow).

### Publish a workflow

After you create or update a workflow in Development, publish it to another environment (typically Production) with [Sync a workflow](/api-reference/workflows/sync-a-workflow). Pass the target environment identifier from the Novu Dashboard.

Use your Development secret key when calling sync. The workflow is copied to the target environment under the same `workflowId`, which is unique in that environment.

<Callout type="info">
Run publish (`sync`) in your CI/CD pipeline after workflow changes are merged. This keeps `Development` environment as the source of truth and makes `Production` environment updates consistent and auditable.
</Callout>

<Tabs items={['Node.js', 'Python', 'cURL']}>
  <Tab value="Node.js">
    ```typescript
    import { Novu } from "@novu/api";

    const novu = new Novu({
      secretKey: process.env.NOVU_SECRET_KEY!,
    });

    export const publishWorkflowToProduction = async () => {
      return await novu.workflows.sync(
        {
          targetEnvironmentId: "target-environment-identifier",
        },
        "payout-initiated"
      );
    };
    ```
  </Tab>
  <Tab value="Python">
    ```python
    import os

    from novu_py import Novu

    with Novu(secret_key=os.environ["NOVU_SECRET_KEY"]) as novu:
        res = novu.workflows.sync(
            workflow_id="payout-initiated",
            sync_workflow_dto={
                "target_environment_id": "target-environment-identifier",
            },
        )
    ```
  </Tab>
  <Tab value="cURL">
    ```bash
    curl -X PUT "https://api.novu.co/v2/workflows/payout-initiated/sync" \
      -H "Authorization: ApiKey ${NOVU_SECRET_KEY}" \
      -H "Content-Type: application/json" \
      -d '{
        "targetEnvironmentId": "target-environment-identifier"
      }'
    ```
  </Tab>
</Tabs>

For request details, refer to [Sync a workflow](/api-reference/workflows/sync-a-workflow).

## Recommended production flow

Follow this sequence when deploying workflow definitions from code:

1. Update with a stable `workflowId`.
2. If update returns **404**, create with that same `workflowId`.
3. If create returns **409**, retry update.
4. Sync to target environments from your CI/CD pipeline.
