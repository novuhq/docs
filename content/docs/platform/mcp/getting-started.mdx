# Getting Started with Novu MCP Server

Connect your AI assistant to Novu's notification infrastructure in under 10 minutes.

## Prerequisites

- **Novu account** with API key from [dashboard.novu.co](https://dashboard.novu.co/settings/api-keys)
- **MCP-compatible AI assistant** (Claude Desktop, Claude.ai, Cursor, Windsurf)
- **Node.js** installed (for Claude Desktop users)

## Step 1: Get Your API Key & Region

<Steps>
<Step title="Access Dashboard">
Visit your [Novu Dashboard Settings](https://dashboard.novu.co/settings/api-keys)
</Step>

<Step title="Copy API Key">
Copy your API key
</Step>

<Step title="Identify Region">
Note your region from the dashboard URL:
- `dashboard.novu.co` = **US region** (use `?region=us` or no parameter)
- `eu.dashboard.novu.co` = **EU region** (use `?region=eu`)
</Step>
</Steps>

## Step 2: Configure Your AI Assistant

<Tabs items={['Claude Desktop', 'Claude.ai', 'Cursor', 'Other Tools']}>
<Tab value="Claude Desktop">

**File Location:**
- **macOS**: `~/Library/Application\ Support/Claude/claude_desktop_config.json`
- **Windows**: `%APPDATA%\Claude\claude_desktop_config.json`

**Configuration:**
```json
{
  "mcpServers": {
    "novu": {
      "command": "npx",
      "args": [
        "mcp-remote",
        "https://mcp.novu.co/mcp?region=us",
        "--header",
        "Authorization:Bearer ${NOVU_API_KEY}"
      ],
      "env": {
        "NOVU_API_KEY": "your-novu-api-key-here"
      }
    }
  }
}
```

**For EU region users:** Change `?region=us` to `?region=eu`

**After configuration:** Restart Claude Desktop completely

</Tab>
<Tab value="Claude.ai">

Navigate to **Settings** → **Integrations** → **+ Add Integration**:

- **Name**: `Novu`
- **URL**: `https://mcp.novu.co/mcp?region=us` (or `?region=eu` for EU)
- **Headers**: `Authorization: Bearer your-novu-api-key`

</Tab>
<Tab value="Cursor">

In **Settings** → **Extensions** → **MCP Servers**, add:

```json
{
  "mcpServers": {
    "novu": {
      "url": "https://mcp.novu.co/mcp?region=us",
      "headers": {
        "Authorization": "Bearer your-novu-api-key"
      }
    }
  }
}
```

</Tab>
<Tab value="Other Tools">

**Standard MCP Configuration:**
- **URL**: `https://mcp.novu.co/mcp?region=us` (or `?region=eu`)
- **Authentication**: Header `Authorization: Bearer your-novu-api-key`
- **Protocol**: Streamable HTTP (MCP over HTTP)

Refer to your AI assistant's MCP documentation for specific setup instructions.

</Tab>
</Tabs>

## Step 3: Test Your Connection

Once configured, test the connection with your AI assistant:

<Steps>
<Step title="Check API Status">
```
"Check my Novu API key status"
```
**Expected response**: `API Key is loaded successfully. Length: 32 characters`
</Step>

<Step title="List Workflows">
```
"Show me all my notification workflows"
```
This confirms you can access your Novu workspace data.
</Step>

<Step title="Find a Subscriber">
```
"Find subscriber with email user@example.com"
```
Replace with an actual email from your workspace.
</Step>
</Steps>

## Troubleshooting Setup

### Common Issues

<Tabs items={['API Key Problems', 'Connection Issues', 'Region Mismatch']}>
<Tab value="API Key Problems">

**"No API key is set"**
- Verify the API key is correctly set in your configuration
- Ensure no extra spaces or characters
- Restart your AI assistant after configuration changes

**"Unauthorized"**
- Check your API key is valid in the Novu Dashboard
- Ensure the key belongs to the correct workspace
- Try regenerating your API key

</Tab>
<Tab value="Connection Issues">

**"Failed to connect"**
- Check your internet connection
- Verify the URL is correct for your region
- For Claude Desktop users: ensure Node.js is installed

**"Server not found"**
- Double-check the server URL
- Verify region parameter is correct
- Remove any trailing slashes from URLs

</Tab>
<Tab value="Region Mismatch">

**Empty results or unexpected data**
- Verify you're using the correct region parameter
- Check your dashboard URL to confirm region
- US: `dashboard.novu.co` = use `?region=us` or no parameter
- EU: `eu.dashboard.novu.co` = use `?region=eu`

</Tab>
</Tabs>
